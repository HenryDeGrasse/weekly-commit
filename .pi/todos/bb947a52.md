## Components
1. `LineageQueryService.java` — recursive CTEs for carry-forward chains, RCDO traversal, scope-change timelines
2. `StructuredEvidenceService.java` — assembles SQL facts + lineage + Pinecone semantic evidence + risk features into typed evidence bundle
3. `StructuredEvidence.java` record — SqlFacts, LineageChain, List<SemanticMatch>, RiskFeatures
4. Update `SemanticQueryService` to use structured evidence for RAG answers
5. Update RAG API response to include evidence metadata
6. Frontend evidence drawer component showing what the AI used

## Key design
- SQL facts from existing repositories (UserWeekFact, CarryForwardFact, etc.)
- Lineage from recursive CTEs on carry_forward_link
- Semantic evidence from existing Pinecone pipeline
- Risk features from existing read model tables
- LLM synthesizes ONLY from the evidence bundle
