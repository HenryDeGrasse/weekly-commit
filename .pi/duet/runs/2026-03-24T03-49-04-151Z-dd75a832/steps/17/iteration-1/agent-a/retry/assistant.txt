#### AI Provider Abstraction (new `ai` package)
- `AiContext` — input record with type constants (TYPE_COMMIT_DRAFT, TYPE_COMMIT_LINT, TYPE_RCDO_SUGGEST, TYPE_RISK_SIGNAL, TYPE_RECONCILE_ASSIST, TYPE_TEAM_SUMMARY)

- `AiProviderRegistry` — manages providers, `ai.enabled` feature flag, graceful degradation (`generateSuggestion` never throws, returns unavailable result on error)

#### Services
- `AiSuggestionService` — stores suggestions (with SHA-256 context hash) and records feedback (ACCEPTED/DISMISSED/EDITED)
- `CommitDraftAssistService` — capability 1: suggest improved title/description/criteria/estimate
- `CommitLintService` — capability 2: rules-based lint with hard (blocks lock) and soft (informational) messages; checks: missing success criteria, duplicate titles, vague titles, parent-level RCDO, estimate inconsistency, over-fragmented plan
- `RcdoSuggestService` — capability 3: suggest RCDO node, only surface above 0.7 confidence threshold, never auto-link
- `RiskDetectionService` — capability 4: 5 risk signals (OVERCOMMIT, UNDERCOMMIT, REPEATED_CARRY_FORWARD, BLOCKED_CRITICAL >48h, SCOPE_VOLATILITY >3 changes); `@Scheduled` daily job at 07:00 UTC for all LOCKED plans
- `ReconcileAssistService` — capability 5: likely outcomes, draft summary, carry-forward recommendations
- `ManagerAiSummaryService` — capability 6: team summary citing RCDO branches, unresolved exceptions, carry-forward patterns, critical blocked items; access-denied for non-managers

#### Controller
- `AiController` — 7 endpoints: POST /api/ai/commit-draft-assist, POST /api/ai/commit-lint, POST /api/ai/rcdo-suggest, GET /api/plans/{id}/risk-signals, POST /api/ai/reconcile-assist, GET /api/teams/{id}/week/{weekStart}/ai-summary, POST /api/ai/feedback
