## What was implemented

### CarryForwardService (`carryforward/service/`)
- **`carryForward(sourceCommitId, targetWeekStart, reason, reasonText, actorUserId)`**: Copies title, description, chess piece, RCDO, estimate and success criteria into a new commit in the target week's plan. Sets `carryForwardSourceId` and increments `carryForwardStreak` (source + 1). New commit placed at bottom priority order. Lazily creates the target plan if missing. If target plan is LOCKED, records a `COMMIT_ADDED` scope-change event (prefixed `CARRY_FORWARD:reason`).
- **`getCarryForwardLineage(commitId)`**: Traverses ancestors (upward via `carryForwardSourceId`) and descendants (downward via `CarryForwardLinkRepository`) and returns the full chain ordered oldest-first.

### TicketService (`ticket/service/TicketService`)
- Full CRUD: create (with auto-generated `T-N` key), read, update (partial), delete.
- **Status workflow**: enforced via a static transition map — BACKLOG → READY/IN_PROGRESS/CANCELED, READY → IN_PROGRESS/BACKLOG/CANCELED, IN_PROGRESS → DONE/BLOCKED/READY/CANCELED, BLOCKED → IN_PROGRESS/CANCELED, DONE/CANCELED are terminals.
- Every status change appends a `WorkItemStatusHistory` entry (with `fromStatus`, `toStatus`, `changedByUserId`).
- Idempotent: same-status transition returns immediately without recording history.

### LinkTicketService (`ticket/service/LinkTicketService`)
- Links a work item to a commit on a DRAFT plan.
- **Duplicate prevention**: checks for other active (non-CANCELED) commits by the same owner linked to the same ticket in the same week.
- **RCDO defaulting**: if commit has no RCDO and ticket has one, the commit inherits it. If both differ, returns a warning message without blocking.
