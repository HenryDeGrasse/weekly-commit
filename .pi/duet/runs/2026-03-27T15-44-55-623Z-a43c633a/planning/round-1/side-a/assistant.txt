## Plan Summary

Here's the reasoning behind the 6-step plan to make AI the core of the product:

### Key Insight
The product already has **comprehensive AI backend infrastructure** (7 AI services, full RAG pipeline, 12 prompt templates, eval framework). The problem is purely **UX posture** — AI is positioned as optional sidebars/buttons rather than the default experience. All 5 shifts reuse existing backend endpoints with zero backend changes.

### The 5 UX Shifts (Steps 1-5)

| Step | Shift | What Changes |
|------|-------|-------------|
| **1** | Auto-run AI lint on save | `AiLintPanel` gets `autoRun` prop, shows on MyWeek automatically for DRAFT plans |
| **2** | Pre-fill reconciliation with AI | Reconcile page auto-calls `reconcileAssist` and pre-populates outcomes with "AI suggested" badges |
| **3** | Expand insights + add manager AI summary | `insightsExpanded` defaults `true`, new `ManagerAiSummaryCard` consumes existing `/ai-summary` endpoint |
| **4** | Freeform "describe → structure" commit creation | New `AiCommitComposer` component: paste a description, AI generates structured commit with chess piece, RCDO, estimates |
| **5** | Proactive risk banners on My Week | New `ProactiveRiskBanner` pushes critical risk signals (overcommit, blocked, carry-forward) to the IC's page |

### Step 6: Integration build check
Ensures all changes compile, lint, typecheck, and pass tests together.

### Design Decisions
- **No backend changes needed** — all endpoints already exist (`commit-draft-assist`, `reconcile-assist`, `ai-summary`, `risk-signals`, `rcdo-suggest`, `commit-lint`)
- **Graceful degradation preserved** — every new component falls back silently when AI is unavailable
- **Existing manual flows preserved** — users can still use manual commit forms, click-to-run lint, etc.
- **Steps are independently verifiable** — each creates self-contained components with tests
