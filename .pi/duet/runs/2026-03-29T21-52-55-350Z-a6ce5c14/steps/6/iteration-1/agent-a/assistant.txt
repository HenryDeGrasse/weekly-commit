## Summary

### `SemanticQueryService.java` changes

**New imports & fields:**
- `ExperimentService`, `HydeService`, `SqlQueryRouter` added as injected dependencies (constructor grows from 10 to 13 params)

**Three new pipeline steps in `query()`:**

**(A) SQL routing check** — inserted after intent classification, before filter/namespace/embedding:
- If `sqlQueryRouter.canHandle(intent, keywords)` → calls `sqlQueryRouter.query(...)` 
- If result is available → returns `RagQueryResult` with `ConfidenceTier.HIGH` immediately (no vector search)
- If SQL query fails → logs and falls through to vector search

**(B) HyDE experiment check** — inserted after namespace resolution, before embedding:
- If `experimentService.isEnabled("hyde")` and value is `"enabled"` → calls `hydeService.generateHypothetical(rewritten, userId)`
- If HyDE available → `textToEmbed = hypotheticalAnswer`; otherwise falls back to `rewritten`
- Used for both dense embedding and sparse encoding in the single-query path

**(C) Experiment-aware answer generation:**
- `aiProviderRegistry.generateSuggestion(ragContext)` → `aiProviderRegistry.generateSuggestion(ragContext, userId)`
- Experiment assignments from the result are recorded in the audit context string via `enrichContextWithExperiments()`

**New private helper:** `enrichContextWithExperiments(contextString, experiments)` — parses the JSON context string and adds `experimentAssignments` field.
