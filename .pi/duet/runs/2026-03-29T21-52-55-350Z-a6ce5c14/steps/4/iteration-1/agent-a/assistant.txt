## Summary of changes

### `AiProvider.java`
Added `default generateSuggestion(AiContext, String modelOverride)` that delegates to `generateSuggestion(context)`. `StubAiProvider` inherits this unchanged.

### `AiSuggestionResult.java`
Added 7th field `Map<String, String> experimentAssignments` to the record. Added backward-compatible 6-param constructor (adds `Map.of()`) and kept the existing 5-param constructor (adds both null promptVersion and `Map.of()`). Updated `unavailable()` factory to pass `Map.of()`.

### `OpenRouterAiProvider.java`
- `generateSuggestion(AiContext)` now delegates to `generateSuggestion(context, null)`.
- New `@Override generateSuggestion(AiContext, String modelOverride)` resolves `effectiveModel` (override or default), passes it to `callOpenRouter` and `parseResponse`.
- `callOpenRouter(systemPrompt, userMessage, effectiveModel)` — uses `effectiveModel` for the request body `model` field and for the `cache_control` Anthropic check.
- `parseResponse` gets a 4-param overload `(body, type, promptVersion, effectiveModel)` that uses `effectiveModel` for `modelVersion` in the result. The existing 3-param overload delegates to it with `this.model`.

### `AiProviderRegistry.java`
- `@Autowired(required=false) ExperimentService experimentService` field.
- New `generateSuggestion(AiContext, UUID userId)` — checks `llm-model` experiment, passes model override on treatment, records `experimentAssignments` in the result.
- `setExperimentService(ExperimentService)` for test wiring.

### Tests
- `OpenRouterAiProviderTest`: added `parseResponse_withModelOverride_usesOverrideForModelVersion` and `parseResponse_threeParamOverload_usesProviderModel`.
- `AiProviderRegistryTest`: added `ExperimentService` mock, and 5 new experiment-aware tests covering: no service wired, control variant, treatment variant, AI disabled, provider throws.
